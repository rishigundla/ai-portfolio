Part G · Automation

Portfolio + Resume Sync

When a project deploys, a GitHub Action updates the portfolio-site and resume-builder repos automatically via pull requests.

1

Project merges to main

Populate apps/<project>/portfolio.meta.json with deployedAt field. Commit to main branch of rishigundla/ai-portfolio.

2

Vercel deploys successfully

Deployment completion triggers the GitHub Action .github/workflows/sync-portfolio.yml.

3

Sync script runs

scripts/sync-deployed.ts reads all portfolio.meta.json files. Authenticates to target repos via PAT.

4

Two pull requests open

PR on rishigundla/portfolio-site: adds project card + case study MDX. PR on rishigundla/resume-builder: adds bullet to all prompt files.

5

Slack notification + manual review

Both PR URLs posted to Slack. Review changes, merge PRs. Portfolio auto-redeploys; resume PDF regenerated via ATS optimizer.
